Markdown 语法教程

轻量级标记语言，让写作更简单优雅

作者：尹肇兴

Markdown 是一种轻量级标记语言，由 John Gruber 于 2004 年创建。它的设计初衷是让人们能够用易读易写的纯文本格式编写文档，然后转换成结构化的 HTML。本教程将系统介绍 Markdown 的核心语法，帮助你快速掌握这一高效写作工具。

一、什么是 Markdown

Markdown 具有以下特点：

  • 简洁易学：语法简单直观，几分钟即可上手
  • 跨平台：纯文本格式，任何文本编辑器都可以打开和编辑
  • 广泛支持：GitHub、GitLab、CSDN、知乎、简书等平台都支持 Markdown
  • 可扩展：支持 HTML 标签，可以实现更复杂的排版效果
  • 版本友好：纯文本格式，非常适合版本控制系统（如 Git）
💡 学习建议：

Markdown 的核心语法非常简单，建议边学边练。你可以使用 Typora、VS Code（配合 Markdown Preview Enhanced 插件）或在线编辑器（如 StackEdit、Dillinger）来实时预览效果。

二、标题

Markdown 支持 6 级标题，使用 # 符号标记：

markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
渲染效果：

一级标题

二级标题

三级标题

四级标题

五级标题
六级标题
💡 提示：

# 与标题文本之间必须有一个空格。为了代码的可读性，建议在标题前后各留一行空行。

另一种标题语法

还可以使用底线形式表示一级和二级标题：

markdown
一级标题
=========

二级标题
---------

三、段落与换行

在 Markdown 中，段落由一个或多个连续的文本行组成，段落之间用一个或多个空行分隔。

段落

只需在段落之间留出空行即可：

markdown
这是第一个段落。

这是第二个段落。

换行

在同一段落内换行，有两种方式：

  • 在行尾添加两个或更多空格，然后按回车
  • 使用 HTML 标签 <br>
markdown
第一行文本（后面有两个空格）
第二行文本

使用 HTML 换行标签：
新的一行

四、强调（字体样式）

Markdown 提供了多种文本强调方式：

效果 语法 快捷键（部分编辑器）
斜体 *文本* 或 _文本_ Ctrl + I
粗体 **文本** 或 __文本__ Ctrl + B
粗斜体 ***文本*** 或 ___文本___ —
删除线 ~~文本~~ —
下划线 <u>文本</u>（HTML 标签） Ctrl + U
高亮 ==文本==（部分编辑器支持） —
综合示例：
这是**粗体**文本，这是*斜体*文本，这是***粗斜体***文本。 这是~~删除线~~文本，这是下划线文本，这是==高亮==文本。

五、列表

Markdown 支持有序列表、无序列表和嵌套列表。

（1）无序列表

使用 -、* 或 + 作为列表标记：

markdown
- 第一项
- 第二项
- 第三项

* 第一项
* 第二项

+ 第一项
+ 第二项
💡 提示：

建议在同一个文档中统一使用一种标记符号，保持风格一致。推荐使用 - 作为无序列表标记。

（2）有序列表

使用数字加英文句点：

markdown
1. 第一项
2. 第二项
3. 第三项
💡 自动编号：

列表标记的数字不会影响渲染结果。下面的代码会渲染成与上面相同的有序列表：

markdown
1. 第一项
1. 第二项
1. 第三项

（3）列表嵌套

在子列表前添加 4 个空格或 1 个制表符（Tab）：

markdown
1. 第一项
    - 子项 A
    - 子项 B
2. 第二项
    - 子项 C
        - 子子项 C1
        - 子子项 C2

（4）列表项包含段落

如果要在列表项中包含段落，需要在段落前添加缩进：

markdown
1.  第一项

    第一项的补充说明段落。

2.  第二项

六、引用

使用 > 符号创建引用块：

markdown
> 这是一段引用文本
> 可以写得很长很长

> 也可以是多层嵌套引用
>> 第二层引用
>>> 第三层引用

引用中包含其他元素

引用块中可以包含 Markdown 其他元素：

markdown
> #### 引用中的标题
>
> 1. 有序列表项一
> 2. 有序列表项二
>
> 这里是**粗体**和*斜体*示例

七、代码块

Markdown 提供了两种代码格式：行内代码和代码块。

（1）行内代码

使用反引号包裹代码：

markdown
这是行内代码示例：`print("Hello World")`

（2）代码块

使用三个反引号包裹代码块，并可指定语言以获得语法高亮：

markdown
```python
def hello():
    print("Hello World")

hello()
```

（3）支持的语言

常见支持语法高亮的语言包括：

语言 标记 语言 标记
Python python JavaScript javascript
C/C++ c / cpp Java java
HTML html CSS css
Bash/Shell bash / shell JSON json
SQL sql YAML yaml

（4）缩进式代码块

另一种创建代码块的方式是使用 4 个空格或 1 个制表符缩进：

markdown
    def hello():
        print("Hello World")
💡 推荐：

推荐使用三个反引号的方式，因为它支持语法高亮和语言标记，可读性更好。

八、链接与图片

（1）超链接

基本语法：[链接文本](URL)

markdown
[GitHub](https://github.com)
[百度搜索](https://www.baidu.com)

带标题的链接

markdown
[GitHub](https://github.com "GitHub 官网")

引用式链接

将 URL 定义在文档其他位置，使正文更简洁：

markdown
这是一个 [GitHub][1] 链接示例。

[1]: https://github.com "GitHub 官网"

自动链接

使用尖括号自动转换为链接：

markdown
<https://github.com>
<example@email.com>

（2）图片

基本语法：![替代文本](图片URL)

markdown
![示例图片](https://example.com/image.png)

带标题的图片

markdown
![示例图片](https://example.com/image.png "图片标题")

引用式图片

markdown
![示例图片][image-ref]

[image-ref]: https://example.com/image.png "图片标题"

图片尺寸控制

Markdown 本身不支持直接调整图片尺寸，需要使用 HTML 标签：

html
<img src="https://example.com/image.png" width="300" height="200" alt="示例图片">
💡 图床推荐：

使用图床服务可以方便地管理和引用图片。常用的图床服务包括：GitHub、Gitee、阿里云 OSS、腾讯云 COS、七牛云等。文中提到的蜜蜂图床（beeimg.cn）也是一个选择。

九、表格

Markdown 表格使用竖线 | 分隔列，使用连字符 - 创建表头与内容分隔线。

（1）基本表格

markdown
| 姓名 | 年龄 | 职业 |
|------|------|------|
| 张三 | 25   | 工程师 |
| 李四 | 30   | 设计师 |
| 王五 | 28   | 产品经理 |

（2）对齐方式

通过在分隔线中使用冒号 : 来设置对齐方式：

markdown
| 左对齐 | 居中对齐 | 右对齐 |
|:--------|:--------:|--------:|
| 内容 A  | 内容 B   | 内容 C  |
| 内容 D  | 内容 E   | 内容 F  |
渲染效果：
左对齐 居中对齐 右对齐
内容 A 内容 B 内容 C
内容 D 内容 E 内容 F

（3）表格内使用其他元素

表格单元格内可以使用文本格式化、链接、代码等：

markdown
| 功能 | 语法 | 示例 |
|------|------|------|
| 粗体 | `**文本**` | **粗体** |
| 斜体 | `*文本*` | *斜体* |
| 链接 | `[文本](URL)` | [GitHub](https://github.com) |

十、分割线

使用三个或更多的连字符 -、星号 * 或下划线 _ 创建分割线：

markdown
三个或更多连字符
---

三个或更多星号
***

三个或更多下划线
___
渲染效果：
💡 提示：

为了与标题语法区分，使用连字符时，建议在分割线前后各留一行空行。

十一、任务列表

任务列表可以创建待办事项清单，常用于项目管理和笔记整理。

（1）基本语法

markdown
- [ ] 未完成任务
- [x] 已完成任务
- [ ] 另一个未完成任务
渲染效果：
  • 未完成任务
  • 已完成任务
  • 另一个未完成任务

（2）嵌套任务列表

markdown
- [x] 学习 Markdown 基础语法
    - [x] 标题
    - [x] 列表
    - [x] 链接和图片
- [ ] 实践练习
    - [ ] 写一篇博客
    - [ ] 整理笔记
💡 注意：

[ ] 和 [x] 中的空格是必需的。任务列表在 GitHub、GitLab、Typora 等平台和编辑器中支持良好。

十二、脚注

脚注用于在文档末尾添加注释或参考文献，使正文更整洁。

（1）基本语法

markdown
这是一个脚注示例[^1]，用于添加额外说明。

[^1]: 这是脚注的内容，会显示在文档末尾。

（2）实际应用

markdown
Markdown 由 John Gruber 创建[^gruber]，并在 GitHub 等平台得到广泛应用[^github]。

[^gruber]: John Gruber 是一位作家和软件开发者，也是 Daring Fireball 博客的创始人。
[^github]: GitHub 是全球最大的代码托管平台，全面支持 Markdown 语法。
💡 兼容性说明：

脚注语法属于 Markdown 扩展语法，不是所有解析器都支持。GitHub Flavored Markdown (GFM)、PHP Markdown Extra 等支持脚注。

十三、高级技巧

（1）转义字符

使用反斜杠 \ 转义特殊字符：

markdown
\* 不是斜体 \*
\# 不是标题
\[ 不是链接

（2）支持 HTML 标签

Markdown 支持内嵌 HTML 标签，可以实现更复杂的格式：

html
<div style="color: red;">红色文本</div>
<span style="font-size: 20px;">大号文本</span>
<details>
<summary>点击展开</summary>
隐藏的内容
</details>

（3）数学公式

部分 Markdown 解析器支持 LaTeX 数学公式：

markdown
行内公式：$E = mc^2$

块级公式：
$$
\int_{a}^{b} f(x) \, dx = F(b) - F(a)
$$

（4）流程图与图表

使用 Mermaid 等扩展可以创建流程图、时序图等：

markdown
```mermaid
graph LR
    A[开始] --> B{判断}
    B -->|是| C[执行 A]
    B -->|否| D[执行 B]
    C --> E[结束]
    D --> E
```

（5）目录生成

部分编辑器支持自动生成目录：

markdown
[TOC]
或
[[toc]]
或
{{ .TOC }}
⚠️ 兼容性提示：

以上高级功能需要特定的 Markdown 解析器或编辑器支持。在使用前请确认目标平台的兼容性。例如，数学公式在 Typora、VS Code（Markdown Preview Enhanced）中支持良好，但 GitHub 原生不支持。
