GitHub SSH Key 配置教程

Windows + Git Bash 环境下的完整配置指南

📖 本文来源：

作者：李智博

原文链接: https://github.com/suyan080515/robcup/blob/main/SSH%20Key%E9%85%8D%E7%BD%AE%E6%95%99%E7%A8%8B.md

📋 适用环境：
  • Windows 10 / Windows 11
  • Git for Windows
  • VS Code
  • GitHub Desktop
  • GitHub 私有/公开仓库

一、为什么使用 SSH？

GitHub 支持两种认证方式：

  • HTTPS：用户名 + Token
  • SSH Key：公钥认证
✅ 推荐使用 SSH 的原因：
  • 不需要每次输入 Token
  • 不依赖 Git Credential Manager
  • VS Code、Git Bash、GitHub Desktop 都可以共用
  • 更稳定，适合长期开发

二、检查是否已有 SSH Key

打开 Git Bash，执行以下命令：

ls -al ~/.ssh

如果已有密钥

会看到以下文件：

id_ed25519
id_ed25519.pub

说明已经有 SSH Key。

如果没有密钥

会提示：

No such file or directory

说明需要重新生成。

三、生成 SSH Key

执行以下命令生成新的 SSH 密钥：

ssh-keygen -t ed25519 -C "你的GitHub邮箱"

随后会出现：

Generating public/private ed25519 key pair.

1. 保存位置

提示：

Enter file in which to save the key
(/c/Users/用户名/.ssh/id_ed25519):
💡 直接按 Enter 即可，使用默认位置。

2. 设置密码（Passphrase）

提示：

Enter passphrase:
💡 推荐直接按 Enter，不设置密码。

随后会再次提示：

Enter same passphrase again:

再次按 Enter。

3. 成功生成

会看到以下输出：

Your identification has been saved in /c/Users/用户名/.ssh/id_ed25519
Your public key has been saved in /c/Users/用户名/.ssh/id_ed25519.pub
The key fingerprint is:
SHA256:...
✅ SSH Key 已生成成功！

四、查看公钥

执行以下命令查看公钥内容：

cat ~/.ssh/id_ed25519.pub

会输出一整行：

ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAI... 你的邮箱@example.com
⚠️ 注意：

复制整行内容，不要换行。

五、添加到 GitHub

按照以下步骤将公钥添加到 GitHub：

1. 打开 GitHub，点击右上角头像

2. 选择 Settings

3. 在左侧菜单找到 SSH and GPG keys

4. 点击 New SSH key

或直接访问：https://github.com/settings/keys

填写信息

  • Title：例如 My Windows PC
  • Key type：保持默认 Authentication Key
  • Key：粘贴刚才复制的公钥内容（ssh-ed25519 A...）

点击 Add SSH key 完成添加。

六、测试 SSH 是否成功

打开 Git Bash，执行以下命令：

ssh -T git@github.com

首次连接

第一次连接会出现：

The authenticity of host 'github.com' can't be established.
Are you sure you want to continue connecting (yes/no/[fingerprint])?
💡 输入 yes 并回车。

成功提示

如果成功，会看到：

Hi 用户名! You've successfully authenticated,
but GitHub does not provide shell access.

例如：

Hi suyan080515! You've successfully authenticated,
but GitHub does not provide shell access.
✅ SSH 配置成功！

七、把仓库改成 SSH

进入你的本地仓库：

cd /d/git/robcup

修改远程仓库地址：

git remote set-url origin git@github.com:用户名/仓库地址.git

查看是否修改成功：

git remote -v

应该显示：

origin  git@github.com:suyan080515/robcup.git (fetch)
origin  git@github.com:suyan080515/robcup.git (push)

八、测试 Git 操作

依次执行以下命令测试：

git fetch
git pull
git push

正常情况下会看到：

Already up to date.
Everything up-to-date
✅ SSH 配置完成！可以正常使用 Git 操作了。

九、常见问题

1. Permission denied (publickey)

问题原因：

GitHub 没有找到你的 SSH Key。

解决方法：

重新执行 cat ~/.ssh/id_ed25519.pub 查看公钥，确认在 GitHub → Settings → SSH and GPG keys 中已经添加。

2. Repository not found

常见原因：
  • Token 失效
  • Credential Manager 缓存错误
  • 登录了错误的 GitHub 账号

说明：使用 SSH 后一般不会再遇到这个问题。

3. 第一次连接提示

如果看到提示：

Are you sure you want to continue connecting?

输入 yes 即可。这是 SSH 首次建立信任关系。

推荐开发环境

VS Code
↓
Git
↓
SSH Key
↓
GitHub

优点：

  • 不依赖浏览器登录
  • 不依赖 Token
  • 不依赖 Credential Manager
  • 更稳定
  • 更适合长期开发

常用 Git 命令

git status          # 查看状态
git pull            # 拉取更新
git push            # 推送提交
git remote -v       # 查看远程仓库
git branch          # 查看当前分支
git log --oneline   # 查看提交历史

配置完成

完成以上步骤后，即可通过 SSH 安全、稳定地访问 GitHub。

💡 建议：

所有新的 GitHub 仓库都使用 SSH 地址进行 Clone、Pull 和 Push。
